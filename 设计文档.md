# Career Coach 智能简历与面试辅导系统

## 一、项目概述

### 技术创新点

本项目的核心创新体现在以下四个方面：

| #    | 创新点                 | 说明                                                         |
| ---- | ---------------------- | ------------------------------------------------------------ |
| ①    | 有状态面试工作流       | 基于 LangGraph 状态机实现「出题→等待回答→追问判断→下一题→生成报告」的全流程自动编排，区别于简单 QA Chain |
| ②    | 用户私有 RAG 知识库    | 支持用户上传行业 JD、面经、技术文档，异步向量化后注入 RAG 上下文，减少 LLM 幻觉，无需每次联网 |
| ③    | LangSmith 全链路可观测 | 接入 LangSmith 追踪每次 Chain 调用的输入输出、Token 消耗与延迟，具备生产级可观测能力 |
| ④    | Redis 驱动高并发架构   | Redis 同时承担 Session 缓存、接口限流、Celery 消息队列，配合 FastAPI 异步框架支撑生产级并发 |

## 二、功能模块规划

### 模块 01 简历分析与优化

| 项目     | 详情                                                         |
| -------- | ------------------------------------------------------------ |
| 功能描述 | 用户上传 PDF简历(word和md文件必须转成pdf格式上传)，系统解析文本内容，基于 LangChain Chain 输出结构化优化建议 |
| 核心功能 | ① 动词替换建议（used→developed，responsible for→led） ② 量化成果缺失检测（缺少数字指标时提示补充） ③ 结构问题诊断（经历描述过短、格式混乱） ④ JD(岗位职责) 差距分析（与目标岗位关键词匹配度对比） |
| 输出格式 | JSON 结构化建议，每条包含：位置定位、问题类型、修改前后对比  |
| 技术实现 | FastAPI 接收文件 → pypdf / python-docx 解析 → LangChain 简历分析 Chain → SSE 流式返回建议 |

### 模块 02 面试模拟与追问

| 项目     | 详情                                                         |
| -------- | ------------------------------------------------------------ |
| 功能描述 | 用户输入目标岗位（含 JD 文本），Agent 自动生成高频面试题，模拟面试官进行多轮追问 |
| 核心功能 | ① 基于 JD + 用户简历生成 5~8 道定制化面试题（分基础/项目/行为三类） ② LangGraph 状态机控制追问逻辑（判断回答完整性，决定是否追问） ③ 每题实时反馈（单题得分 + 改进建议，SSE 流式输出） ④ 全程结束后生成结构化评估报告 |
| 状态流转 | 初始化 → 出题 → [等待回答] → 追问判断 → (追问循环) → 下一题 → 报告生成 |
| 技术实现 | LangGraph StateGraph + ToolNode，FastAPI WebSocket 或 SSE，前端 Vue 双栏布局 |

### 模块 03 可视化反馈报告

| 项目     | 详情                                                         |
| -------- | ------------------------------------------------------------ |
| 功能描述 | 面试完成后生成包含图表的结构化评估报告，左侧展示逐题反馈，右侧流式输出整体报告 |
| 图表内容 | ① 雷达图（ECharts）：多维度评分（数据结构/系统设计/并发编程/数据库/网络） ② 横向进度条（CSS 动画）：综合得分 / 表达逻辑 / 技术深度 / 项目经验 ③ 逐题得分卡片：单题 100 分制 + 文字点评 |
| 流式体验 | 后端 FastAPI StreamingResponse + 前端 EventSource，报告按节逐步渲染（亮点→建议→总评） |
| 交互设计 | 左侧题目列表可点击展开单题详情；右侧报告支持导出 PDF；报告数据持久化存入 MySQL |
| 技术实现 | ECharts vue-echarts + CSS transition 动画 + SSE 分段流式推送 |

### 模块 04 知识库管理 & Agent 操作

| 项目       | 详情                                                         |
| ---------- | ------------------------------------------------------------ |
| 功能描述   | 用户上传文件构建私有知识库，供 RAG 检索；LangGraph Agent 可代替用户执行页面操作 |
| 知识库功能 | ① 支持上传 PDF格式文件 ② 后台异步（Celery）切片 → Embedding → 存入向量数据库 ③ 知识库文件管理（列表、删除、状态查看） ④ 检索时自动从私有知识库 Top-K 召回注入 Prompt |
| Agent 操作 | 用户对话触发 Agent（如「帮我把简历针对这个 JD 优化一遍」）→ Agent 调用内部 Tool：拉取简历→分析 JD→执行优化 Chain→写回简历草稿，前端通过 SSE 展示 Agent 步骤进度 |
| 技术实现   | LangGraph ToolNode + FastAPI 内部 Tool 注册 + Celery 异步任务队列 + Redis Stack |

## 三、系统架构设计

### 3.1 整体架构分层

系统采用经典五层架构，各层职责清晰，便于水平扩展。

| 层级      | 技术选型                                   | 核心职责                                      | 高并发策略                            |
| --------- | ------------------------------------------ | --------------------------------------------- | ------------------------------------- |
| 前端层    | Vue 3 + Vite + Pinia ECharts / vue-echarts | 用户交互、简历编辑、面试对话、报告可视化      | EventSource SSE 流式接收 本地状态管理 |
| 网关层    | Nginx                                      | 反向代理、SSL 终止、静态资源、限流（令牌桶）  | 连接数限制 upstream 负载均衡          |
| 业务层    | FastAPI (asyncio) Celery Worker            | 用户认证、简历服务、面试服务、文件服务        | 全异步 I/O 多进程部署（Gunicorn × 4） |
| AI 编排层 | LangChain LangGraph LangSmith              | RAG 检索、Prompt 组装、Chain 执行、Agent 编排 | 流式生成（astream） 链路追踪监控      |
| 数据层    | MySQL 8 Redis 7 阿里云 OSS                | 结构化数据、缓存/队列/向量检索、文件存储      | 连接池（asyncpg） Redis 缓存穿透防护  |

### 3.2 技术选型详细说明

#### 前端技术栈

| 技术            | 说明                                                         |
| --------------- | ------------------------------------------------------------ |
| Vue 3 + Vite    | Composition API + `<script setup>` 语法，Vite 开发热更新速度快，适合生产环境构建 |
| Pinia           | 轻量级状态管理，替代 Vuex，支持 TypeScript，管理用户信息、面试状态、报告数据 |
| ECharts         | 阿里开源图表库，`vue-echarts` 封装，setOption 支持数据增量更新，完美配合流式报告渲染 |
| EventSource SSE | 浏览器原生 API，无需额外依赖，接收后端流式推送，实现打字机文字效果和实时图表更新 |
| pdf.js          | Mozilla 开源 PDF 渲染库，在线预览用户上传的简历文件          |
| marked.js       | Markdown 渲染，LLM 输出的 Markdown 格式建议文本转 HTML 显示  |

#### 后端技术栈

| 技术           | 说明                                                         |
| -------------- | ------------------------------------------------------------ |
| FastAPI        | Python 异步 Web 框架，基于 asyncio，原生支持 OpenAPI 文档，StreamingResponse 实现 SSE |
| SQLAlchemy 2.0 | 异步 ORM（async session），配合 asyncpg 驱动连接 MySQL，避免阻塞事件循环 |
| Alembic        | 数据库迁移工具，版本化管理 Schema 变更，生产环境必备         |
| Celery + Redis | 异步任务队列，文件切片/Embedding 等耗时任务异步执行，Redis 作为 Broker 和 Result Backend |
| JWT + OAuth2   | FastAPI 内置 OAuth2PasswordBearer，JWT Token 认证，Redis 存储黑名单 |
| Pydantic v2    | 请求/响应数据校验，自动生成接口文档，与 FastAPI 深度集成     |

#### AI 层技术栈

| 技术           | 说明                                                         |
| -------------- | ------------------------------------------------------------ |
| LangChain      | 核心 RAG 框架：Document Loader → Text Splitter → Embedding → VectorStore → Retriever → Chain |
| LangGraph      | 有状态 Agent 编排：StateGraph 定义节点和边，支持条件分支（如追问判断）、循环（追问循环） |
| LangSmith      | LLM 链路追踪与评估平台，自动记录每次 Chain 调用的 Prompt、输出、Token 用量和延迟 |
| Embedding 模型 | OpenAI text-embedding-3-small 或本地 BGE 模型，将文本转为向量用于相似度检索 |
| 向量数据库     | Redis Stack（RediSearch 模块），存储文档向量，支持 Top-K 语义检索 |

#### 数据层技术栈

| 技术     | 说明                                                         |
| -------- | ------------------------------------------------------------ |
| MySQL 8  | 主数据库，存储用户信息、简历版本、对话历史、面试报告，InnoDB 引擎支持事务 |
| Redis 7  | 四重职责：① JWT Session 缓存（TTL 自动失效）② 接口限流（滑动窗口计数器）③ Celery 消息队列 ④ 向量检索（RediSearch） |
| 阿里云 OSS | 兼容 S3 协议的对象存储，存储用户上传的简历文件和知识库文件，高可用免运维 |
### 3.3 高并发设计方案

系统面向生产环境设计，高并发保障来自以下四个层面：

| 层面     | 方案                                                         | 效果                                     |
| -------- | ------------------------------------------------------------ | ---------------------------------------- |
| 接口层   | Nginx 令牌桶限流 + FastAPI 全异步 I/O + Gunicorn 多 Worker   | 单服务器支撑数百并发请求，I/O 等待不阻塞 |
| 缓存层   | Redis 缓存热点数据（用户信息、简历摘要），TTL 自动刷新       | 减少 80% 以上 MySQL 读压力               |
| 任务队列 | Celery + Redis：文件向量化等耗时任务异步处理，不占用主线程   | 接口响应时间 < 200ms，任务后台执行       |
| LLM 限流 | Redis 滑动窗口：每用户每分钟限制 LLM 调用次数，防止 API 超限 | 保护上游 LLM API，避免 429 错误          |

## 五、生产部署架构

### 5.1 Docker Compose 服务编排

全部服务通过 Docker Compose 一键编排，开发与生产环境一致，便于毕设答辩演示：

| 服务名   | 镜像                   | 端口      | 说明                      |
| -------- | ---------------------- | --------- | ------------------------- |
| nginx    | nginx:alpine           | 80 / 443  | 反向代理，SSL 终止，限流  |
| web      | 自建 Vue 镜像          | —         | Nginx 直接 serve 静态文件 |
| api      | 自建 FastAPI 镜像      | 8000      | Gunicorn × 4 Worker       |
| celery   | 同 api 镜像            | —         | Celery Worker × 2         |
| mysql    | mysql:8.0              | 3306      | 主数据库，挂载 Volume     |
| redis    | redis/redis-stack:latest | 6379/8001 | 缓存 + 消息队列 + 向量检索 |
| minio    | minio/minio:latest     | 9000/9001 | 对象存储 + Web 控制台（开发环境，生产使用阿里云 OSS） |

### 5.2 部署拓扑

生产环境流量路径：Internet → Nginx（443/SSL） → FastAPI（多进程） → Redis（限流+队列+向量检索） → Celery Worker → 数据层（MySQL + 阿里云 OSS）

LangSmith 为旁路监控，不在请求链路中，通过 SDK 异步上报 Trace 数据，不影响主链路性能。
