# 数据库设计

## 用户表 (users)

| 字段名 | 类型 | 默认值 | 备注 | 索引 |
|--------|------|--------|------|------|
| id | BIGINT UNSIGNED | AUTO_INCREMENT | 主键，用户 ID | PRIMARY |
| username | VARCHAR(50) | - | 登录用户名，唯一 | UNIQUE |
| password | VARCHAR(255) | - | 加密后的密码 | - |
| nickname | VARCHAR(50) | '' | 昵称/显示名称 | - |
| avatar_url | VARCHAR(500) | '' | 头像 OSS 地址 | - |
| status | TINYINT | 1 | 状态：1=正常, 0=禁用 | INDEX |
| created_at | DATETIME | CURRENT_TIMESTAMP | 创建时间 | INDEX |
| updated_at | DATETIME | CURRENT_TIMESTAMP ON UPDATE | 更新时间 | - |
| last_login_at | DATETIME | NULL | 最后登录时间 | - |

## 对话表 (conversations)

| 字段名 | 类型 | 默认值 | 备注 | 索引 |
|--------|------|--------|------|------|
| id | BIGINT UNSIGNED | AUTO_INCREMENT | 主键，对话 ID | PRIMARY |
| user_id | BIGINT UNSIGNED | - | 所属用户 ID，外键 users.id | INDEX |
| title | VARCHAR(100) | '新对话' | 对话标题 | - |
| message_count | INT UNSIGNED | 0 | 消息总数（冗余字段，加速列表查询） | - |
| last_message_at | DATETIME | NULL | 最近一条消息的时间 | INDEX |
| created_at | DATETIME | CURRENT_TIMESTAMP | 创建时间 | - |
| updated_at | DATETIME | CURRENT_TIMESTAMP ON UPDATE | 更新时间 | - |

复合索引：`(user_id, updated_at DESC)`，用于"按用户查对话列表 + 按更新时间倒序"。

## 消息表 (messages)

| 字段名 | 类型 | 默认值 | 备注 | 索引 |
|--------|------|--------|------|------|
| id | BIGINT UNSIGNED | AUTO_INCREMENT | 主键，消息 ID | PRIMARY |
| conversation_id | BIGINT UNSIGNED | - | 所属对话 ID，外键 conversations.id | INDEX |
| role | VARCHAR(20) | - | 角色：user / assistant / system | - |
| content | TEXT | - | 消息内容 | - |
| created_at | DATETIME | CURRENT_TIMESTAMP | 创建时间 | - |

复合索引：`(conversation_id, created_at)`，用于"按对话查消息 + 按时间正序"。

## 简历表 (resumes)

| 字段名 | 类型 | 默认值 | 备注 | 索引 |
|--------|------|--------|------|------|
| id | BIGINT UNSIGNED | AUTO_INCREMENT | 主键，简历 ID | PRIMARY |
| user_id | BIGINT UNSIGNED | - | 所属用户 ID | INDEX |
| filename | VARCHAR(255) | - | 原始文件名 | - |
| file_url | VARCHAR(500) | - | OSS 文件访问地址 | - |
| target_jd | TEXT | NULL | 目标岗位 JD 文本（可选） | - |
| suggestions | JSON | - | 结构化优化建议列表 | - |
| total_issues | INT UNSIGNED | 0 | 问题总数（冗余字段，加速列表查询） | - |
| created_at | DATETIME | CURRENT_TIMESTAMP | 创建时间 | - |

复合索引：`(user_id, created_at DESC)`，用于"按用户查简历列表 + 按时间倒序"。

**suggestions 字段示例**：

```json
[
  {
    "type": "verb_replacement",
    "location": "工作经历 - 项目经历第 1 段",
    "issue": "使用了弱动词 used",
    "before": "Used React to build the page",
    "after": "Developed the page using React"
  }
]
```

**type 枚举**：`verb_replacement` / `missing_quantification` / `structure` / `jd_gap`
