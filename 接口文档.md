# API 接口文档

> 基准路径：`http://localhost:8000`
>
> 响应格式：统一使用 `ApiResponse` 结构

---

## 鉴权说明

测试阶段使用请求头 `X-User-Id` 传递当前用户身份，后续接入 JWT 后切换为 `Authorization: Bearer <token>`，业务代码改动量较小。

| 阶段 | 请求头 | 示例 |
|------|--------|------|
| 当前（测试） | `X-User-Id` | `X-User-Id: 1` |
| 后续（接入 JWT） | `Authorization` | `Authorization: Bearer eyJ...` |

**需要鉴权的接口**：除 `1.1 注册` / `1.2 登录` 外，其余所有接口都需要在请求头中携带 `X-User-Id`。

**未携带或非法 user_id 的错误响应**：

```json
{
  "code": 401,
  "message": "未登录或用户信息无效",
  "data": null
}
```

---

## 通用响应结构

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 业务状态码，200 表示成功 |
| message | string | 提示信息 |
| data | any | 返回数据，可为对象、数组、null |

---

## 业务状态码

| 状态码 | 说明 |
|--------|------|
| 200 | 请求成功 |
| 400 | 请求参数错误 |
| 401 | 未登录 / 用户名或密码错误 |
| 403 | 无权访问该资源（如访问他人对话） |
| 404 | 资源不存在 |
| 409 | 资源冲突（如用户名已存在） |
| 422 | 参数校验不通过 |
| 500 | 服务器内部错误 |

## 1. 用户模块

### 1.1 用户注册

**接口描述**

新用户注册，成功后返回用户信息（不含密码）。

**请求**

```
POST /api/auth/register
Content-Type: application/json
```

**请求体参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | string | 是 | 登录用户名，3-50 字符，全局唯一 |
| password | string | 是 | 密码，6-128 字符 |
| nickname | string | 否 | 昵称/显示名称，不传则使用 username |

**请求体示例**

```json
{
  "username": "zhangsan",
  "password": "abc123456",
  "nickname": "张三"
}
```

**成功响应**

```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "id": 1,
    "username": "zhangsan",
    "nickname": "张三",
    "avatar_url": "",
    "status": 1,
    "created_at": "2026-06-26T12:00:00",
    "last_login_at": null
  }
}
```

**错误响应示例**

```json
{
  "code": 409,
  "message": "用户名已存在",
  "data": null
}
```

```json
{
  "code": 422,
  "message": "密码长度不能少于 6 位",
  "data": null
}
```

## 2. 对话模块

### 2.1 新建对话

**接口描述**

创建一个新的对话会话，返回会话 ID。

**请求**

```
POST /api/conversation/create
Content-Type: application/json
X-User-Id: 1
```

**请求体参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| title | string | 否 | 对话标题，不传默认为"新对话" |

**请求体示例**

```json
{
  "title": "Java 面试题练习"
}
```

**成功响应**

```json
{
  "code": 200,
  "message": "创建成功",
  "data": {
    "id": 10,
    "user_id": 1,
    "title": "Java 面试题练习",
    "created_at": "2026-06-27T14:30:00",
    "updated_at": "2026-06-27T14:30:00"
  }
}
```

---

### 2.2 发送消息(先调通,后期再弄上下文的事儿)

**接口描述**

向指定对话发送一条消息，AI 回复后一并入库并返回。

**请求**

```
POST /api/conversation/message/{conversation_id}
Content-Type: application/json
X-User-Id: 1
```

**路径参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| conversation_id | int | 是 | 对话 ID |

**请求体参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| content | string | 是 | 用户发送的消息内容 |

**请求体示例**

```json
{
  "content": "什么是 JVM 内存模型？"
}
```

**成功响应**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "user_message": {
      "id": 21,
      "conversation_id": 10,
      "role": "user",
      "content": "什么是 JVM 内存模型？",
      "created_at": "2026-06-27T14:32:00"
    },
    "ai_message": {
      "id": 22,
      "conversation_id": 10,
      "role": "assistant",
      "content": "JVM 内存模型（JMM）是 Java 虚拟机规范定义的...",
      "created_at": "2026-06-27T14:32:03"
    }
  }
}
```

**错误响应示例**

```json
{
  "code": 404,
  "message": "对话不存在",
  "data": null
}
```

```json
{
  "code": 500,
  "message": "AI 服务异常，请稍后重试",
  "data": null
}
```

### 2.3 对话列表

**接口描述**

按更新时间倒序返回当前用户的对话列表。

**请求**

```
GET /api/conversation/list?page=1&page_size=20
X-User-Id: 1
```

**查询参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| page | int | 否 | 页码，默认 1 |
| page_size | int | 否 | 每页数量，默认 20，最大 100 |

**成功响应**

```json
{

  "message": "success",
  "data": {
    "total": 5,
    "page": 1,
    "page_size": 20,
    "items": [

        "created_at": "2026-06-27T14:30:00",
        "updated_at": "2026-06-27T15:10:00"
      },
      {
        "id": 9,
        "title": "简历优化建议",
        "message_count": 4,
        "last_message_at": "2026-06-26T20:00:00",
        "created_at": "2026-06-26T19:30:00",
        "updated_at": "2026-06-26T20:00:00"

    ]
  }
}
```
